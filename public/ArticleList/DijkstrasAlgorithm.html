How Dijkstra's Algorithm (Shortest Path) Works?

Dijkstra's Algorithm is a famous greedy algorithm used to solve the shortest path problem for a graph with non-negative edge weights. It computes the shortest path from a starting node (the "source") to all other nodes in the graph.

The algorithm works by selecting the node with the smallest known distance, updating the distances of its neighbors, and repeating this process until all nodes have been visited. This ensures that the shortest path to each node is determined step by step, making it an efficient solution.

Steps of Dijkstra's Algorithm:

  1. Set the distance to the source node as 0 and to all other nodes as infinity.
  2. Mark the source node as visited. For all its neighbors, calculate the tentative distance using the current nodeâ€™s distance. If this new distance is smaller than the previously known distance, update it.
  3. Move to the unvisited node with the smallest tentative distance and repeat the process of updating distances for its neighbors.
  4. Continue this process until all nodes are visited, ensuring that the shortest path to each node is found.

Example: Consider a graph where nodes represent cities and edges represent the distance between them. Starting from city A, Dijkstra's Algorithm will calculate the shortest distance to all other cities (nodes), considering the sum of edge weights (distances) in each step.

Time Complexity: The time complexity of Dijkstra's Algorithm depends on the implementation. Using a priority queue (min-heap), the complexity is O((V + E) log V), where V is the number of vertices (nodes) and E is the number of edges in the graph.

Limitations:* Dijkstra's Algorithm does not work with graphs that have negative edge weights because the greedy approach of choosing the closest node might not always lead to the optimal solution in such cases.
